# GP-CATE

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)
[![Python 3.9+](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/downloads/)

Reference implementation and full reproduction code for

> **Calibrated Inference for the Conditional Average Treatment Effect in
> the Few-Placebo Regime via Gaussian Processes**
> Eichi Uehara (AFLO). [[paper PDF](docs/paper.pdf)]

**GP-CATE** estimates the conditional average treatment effect (CATE)
`tau(x) = E[Y(1) - Y(0) | X = x]` **with calibrated credible intervals**
in the *few-placebo regime*, where one treatment arm is much smaller
than the other (`N0 << N1`) — unequal-allocation trials,
budget-constrained A/B tests, rare-regime studies.

The method models each arm's outcome surface with a Gaussian process
and reads the CATE off the difference of the two arm posteriors:

```
tau(x) | D  ~  N( m1(x) - m0(x),  s1(x)^2 + s0(x)^2 )
```

Because the GP posterior is calibrated by construction — wide where an
arm's units are sparse, narrow where they are dense — the scarce
control arm's uncertainty enters the posterior **directly**, rather than
as the unmodelled bias that makes meta-learner intervals under-cover.

## The paper's three claims, and where each is reproduced

| Claim | Section | Table | Script |
|---|---|---|---|
| **Diagnosis** — a Bayesian X-Learner under-covers in the few-placebo regime, from *first-order nuisance bias* | §3 | Table 1 | [`experiments/table1_bias_diagnosis.py`](experiments/table1_bias_diagnosis.py) |
| **Negative result** — the doubly-robust / orthogonal score does *not* repair it (overlap amplification + an efficiency/unbiasedness conflict), even with the propensity known exactly | §4 | Table 2 | [`experiments/table2_negative_dr.py`](experiments/table2_negative_dr.py) |
| **Method** — GP-CATE is calibrated where Causal Forest and BART are not, across three designs | §5 | Tables 3–4 | [`experiments/table3_4_coverage.py`](experiments/table3_4_coverage.py) |
| — point accuracy (calibration is not bought with error) | §5 | Table 5 | [`experiments/table5_pehe.py`](experiments/table5_pehe.py) |
| — posterior contraction, arm-variance split, treated-arm subsampling | §5 / App. B | — | [`experiments/diagnostics.py`](experiments/diagnostics.py) |

The headline result (coverage of nominal-95% intervals, averaged over
the `N0` sweep):

| design | Causal Forest | BART | **GP-CATE** |
|---|---|---|---|
| linear      | 0.79 | 0.93 | **0.98** |
| non-linear  | 0.61 | 0.82 | **0.95** |
| IHDP        | 0.40 | 0.70 | **0.96** |

GP-CATE is the only estimator calibrated throughout; the cost is
honestly wide intervals on high-dimensional covariates (IHDP), which
report the true difficulty of the problem rather than concealing it.

## Install

```bash
git clone https://github.com/EichiUehara/GP-CATE.git
cd GP-CATE
python -m venv venv && source venv/bin/activate

# Core: GP-CATE + the diagnosis/negative-result experiments
pip install -e .

# Optional: the interval baselines (Causal Forest via econml, BART via
# bartz/JAX) used in Tables 3-5. Heavy.
pip install -e ".[baselines]"
```

Core dependencies are `numpy`, `scipy`, `scikit-learn`, `pandas`. The
baselines add `econml` and `bartz`. GP-CATE itself uses only
scikit-learn's exact Gaussian process — no JAX, PyMC, or specialised GP
library required.

## Quick start

```python
import numpy as np
from gpcate import GPCATELearner

# X: (n, d) covariates, W: (n,) binary treatment, Y: (n,) outcome
rng = np.random.default_rng(0)
X = rng.standard_normal((530, 4))
W = np.r_[np.zeros(30), np.ones(500)].astype(int)   # few-placebo: 30 vs 500
Y = 0.5 * X[:, 0] + W * (1 + 0.3 * X[:, 0]) + 0.5 * rng.standard_normal(530)

model = GPCATELearner().fit(X, W, Y)
tau, lo, hi = model.credible_interval(X, alpha=0.05)   # 95% credible interval
```

`GPCATELearner.predict` returns the posterior mean, the credible bounds,
and the posterior standard deviation of `tau(x)`.

## Reproduce the paper

Every number in the paper is regenerated by:

```bash
python -m experiments.reproduce_all            # full protocol (paper settings)
python -m experiments.reproduce_all --quick    # fast smoke run (few seeds)
```

or one table at a time:

```bash
python -m experiments.table1_bias_diagnosis    # Table 1  (no baselines needed)
python -m experiments.table2_negative_dr       # Table 2  (no baselines needed)
python -m experiments.table3_4_coverage        # Tables 3-4 (needs baselines)
python -m experiments.table3_4_coverage --only ihdp
python -m experiments.table5_pehe              # Table 5    (needs baselines)
python -m experiments.diagnostics              # contraction / arm split / subsampling
```

Each script writes a CSV to `experiments/results/` and prints a table
to stdout, alongside the paper's published numbers for comparison.
`--quick` runs a handful of seeds for a fast end-to-end check; the full
protocol (Appendix B: 20 seeds on the synthetic designs; 10 IHDP
replications × 4 control sub-samples) reproduces the published values.

## Designs

All three designs have a *known* `tau(x)`, so coverage is measurable
(`gpcate/datasets.py`):

- **linear** — `X ~ N(0, I_4)`, logistic propensity in `X0`, linear
  `tau = 1 + 0.3 X0 - 0.2 X2`.
- **non-linear** — two-covariate propensity, interaction-laden response
  surface, `tau = 1 + sin(1.4 X0) - 0.3 X2^2`.
- **IHDP** — the semi-synthetic benchmark of Hill (2011): 25 real
  covariates, 747 units, simulated response surface. Ten replications
  are committed under [`data/ihdp/`](data/ihdp/). The few-placebo regime
  is induced by keeping all treated units and sub-sampling the controls.

## Repository layout

```
gpcate/
  gp_cate.py        GP-CATE: Bayesian T-learner with GP arms (the method)
  pseudo_outcome.py X-Learner / DR-score Bayesian regression (§3-4 diagnosis)
  meta_learners.py  S / T / X / DR point-estimate meta-learners (Table 5)
  baselines.py      Causal Forest (econml) and BART (bartz) interval baselines
  datasets.py       linear / non-linear / IHDP designs + few-placebo sampling
  basis.py          polynomial / RFF / linear feature bases
experiments/        one script per table + reproduce_all orchestrator
data/ihdp/          ten IHDP replications (CSV) + provenance
tests/              unit tests (run with: pytest)
docs/paper.pdf      the paper
```
